# Zijiang Attachment Downloader

## Overview

读取抽凭结果 Excel，提取每行用于定位网页记录的关键字段，自动登录目标网站，检索对应记录，下载附件，并输出下载结果清单与失败日志。

优先把这个 skill 当成“批量执行器”：输入一个 Excel 文件和下载目录，得到一批附件文件与结果报告。不要把账号密码暴露到对外输出中。

## Workflow

1. 确认输入参数：Excel 路径、sheet、表头行、数据起始行、下载目录。
2. 读取 Excel，标准化字段名和单元格格式。
3. 为每一行生成检索键，优先使用高置信字段组合。
4. 登录网站并进入目标数据列表/明细页面。
5. 对每一行执行“检索 → 定位 → 打开记录 → 下载附件”。
6. 记录成功、失败、跳过、重名处理结果。
7. 输出结果 CSV/JSON 日志，必要时回写 Excel。

## Record Matching Strategy

按以下优先级组合匹配网页记录；若高优组合找不到，再降级：

### 优先级 A

- 凭证编号
- 凭证日期
- 借方发生金额 或 贷方发生金额
- 摘要

### 优先级 B

- 凭证日期
- 科目编号
- 科目名称
- 对方科目
- 金额（借/贷之一）

### 优先级 C

- 附件索引号
- 附件内容填写
- 抽凭名称 / 关联抽凭名称

## Data Normalization Rules

执行前先标准化：

- 日期统一为 `YYYY-MM-DD`。
- 金额去掉千分位逗号，转成十进制字符串；比较时允许格式差异，但数值必须一致。
- 空值统一成空字符串。
- 文本字段去首尾空格，多个空白折叠为一个空格。
- 如果“借方发生金额”为 0，则优先使用“贷方发生金额”；反之亦然。
- “附件索引号”可包含多个值（如逗号分隔），需要拆分并逐个比对或用于文件命名。

## Download Rules

默认规则：下载目标记录下的全部附件。

若用户明确要求，可改成：

- 仅下载凭证附件
- 按附件名称关键词过滤
- 按附件索引号过滤

下载文件建议按以下目录结构保存：

- `{下载目录}/{抽凭名称或关联抽凭名称}/{凭证日期}_{凭证编号}_{序号}/`

文件重名时：

- 保留原文件名
- 若冲突，追加 `_2`、`_3` 等序号

## Outputs

至少输出：

1. `download-results.csv`
   - 序号
   - 凭证编号
   - 凭证日期
   - 匹配状态
   - 下载状态
   - 下载文件数
   - 下载目录
   - 失败原因

2. `download-results.json`
   - 保留更完整的调试信息、页面定位信息、实际文件名

如用户要求，可额外回写 Excel，新增列例如：

- 下载状态
- 下载文件数
- 下载目录
- 失败原因
